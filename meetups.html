Meetups

October 24th, 2017

Meetup Information

The first meetup I went to was the DenverScript meetup on October 24th, at CodeTalent. The meeting covered Chrome Dev tools (Chrome 60) and CSS Variables.

People I Met

I met James Sullivan, a recent Turing Web Dev graduate. James is on the job hunt and optimistically so. James cited the meetups as a great place to network, pick up new skills and score free meals.

I first met Guillermo Moratorio on the 4th floor Platte balcony and then again at this meetup. Guillermo is a Galvanize graduate and works as software developer at Voalte. Super nice guy!

What I Learned

I learned that CSS variables (custom properties) permit the declaration and usage of cascading variables in stylesheets. This is considered experimental but is 78% globally supported.

I learned that I was able to understand the syntax in the lightning talks for the most part.

I learned that the Chrome dev tools are an extremely helpful tool that is constantly being updated. The talk was on Chrome 60 and 61 has already been released. The updates make it easier to work directly in the console when debugging and editing code.

Objects in the console now display more information than before.

Questions I Have

  • Will we be learning about CSS Variables at Galvanize/is it a "game changer"?
  • Is it best practice to use the dev tools for all debugging etc.?
  • The dev tools have SO MANY different menus etc., what are the main functions to learn?

Meetups

November 1st, 2017

Meetup Information

The third meetup I attended was about transitioning to Java 8 from Java 7. The meetup was hosted by Women Who Code at Galvanize, Platte campus on Nov. 1st.

People I Met

I first met Nowai Matthew, the director of WWC in Denver. She stated that she was a front-end dev but she had some good input and seemed to follow the Java presentation without issue. Nowai works as an Application Engineer for Lyons Consulting Group.

The presenter was Laura Patterson. She had a lot of valuable insight on what it's like to move to a new version of software within a corporate structure. Laura had a lot of great info and she was able to convey most of it clearly and concisely. Laura works as a Java software engineer for AppNexus.

What I Learned

With Java being way out of my current league, I mainly learned about the greater scope of of how companies deal with implementing upgrades and how as an individual you should never become complacent with your current skillset.

I learned that you may end up working at a place that does not require you to learn new skills to complete the day to day. This means that it is very easy to become stagnant with your skillset if you don't constantly push yourself to evolve. Some of the benefits of pushing yourself to learn the new tools is that you can present a case to your employer why they could/should upgrade if they havenâ€™t on their own.

I learned that upgrades my be "shiny and new" but most of the time, a large company will prefer stability over shiny new features.

I also learned that i was only allowed to attend this meetup because i came with a women; which the director of the group decided was important to announce.

Questions I Have

  • This information presented here was all geared toward back-end development and others I have seen were geared towards front-end. Do the majority of people fall into either one as opposed to being full-stack?
  • Is Java considered to be a modern standard for most companies?
  • Will we be learning Java during the WDI at Galvanize?
