Learning to Learn

Oct.22nd

git/github October 22, 2017 I wanted to spend some more time with git/github. I watched a 4 -part series called Github for noobs. I picked up some nice git commands and gained a better understanding of github. Set the Stage When i set the stage, the first thing I do is mute notifications on my computer and if my phone is around, I mute and place it facedown. Next, I open a new browser window and then split the screen with my terminal. In my terminal, I will cd to where I need to be and then command+k to clear the history and start with a blank terminal. If I am not watching videos/podcasts, I will play some background music. Finally, I wil usualy start the pomodoro timer I have to keep me grounded for 25 minutes minimum. Setting the stage helped me to prepare myself mentally for a new task by removing all distractions and narrowing my focus on the subject at hand. Practicing music my whole life, I have been very familiar with setting the stage etc. so I can't say my process in this regard has changed that drastically. Reflection I learned a few new git commands as well as how to create, delete, and switch branches in github through the terminal. My favorite command I found was git commit -am “” which will add(stage) and commit all files in one step. I also picked up a nice mindset for learning from the series. Being a "noob" at something is to be embraced as the pressure on yourself is much less if you remember that you are at the beginning of the learning process so mistakes will and have ot be made. They are good in fact! I also found that reading through the comments on the videos I watched would reveal some other great info from the community.
